Synthetic Data Generator

AI-based system for generating realistic synthetic datasets

1. Project Explanation

1.1 Introduction

In today's data-driven world, access to high-quality datasets is essential for building, testing, and validating machine learning models. However, real-world data is often limited due to privacy concerns, regulatory constraints, or availability issues. This creates a significant barrier for data scientists, researchers, and developers who need reliable data to experiment and innovate.

Synthetic data has emerged as a powerful solution to this problem. By generating artificial datasets that preserve the statistical properties of real data, it becomes possible to simulate realistic scenarios without exposing sensitive information.

This project focuses on developing a flexible and scalable synthetic data generation system that allows users to create customized datasets based on their specific needs. The system aims to support both numerical and textual data generation while maintaining realism, usability, and efficiency.

1.2 Problem Statement

Despite the growing demand for data in machine learning and analytics, accessing high-quality and usable datasets remains a major challenge. Many datasets are either restricted due to privacy regulations, insufficient in size, unbalanced, or not representative of real-world scenarios.

Additionally, generating test data for applications often requires significant manual effort, which can be time-consuming and inefficient. Existing solutions are either too complex, lack flexibility, or fail to produce realistic data distributions.

This creates a gap for a user-friendly system that can generate synthetic data while preserving statistical integrity and allowing customization based on user-defined parameters.

1.3 Objective

The main objective of this project is to design and develop a synthetic data generation system that enables users to easily create realistic and customizable datasets. Specifically, the system aims to:

  • Allow users to define the number of rows and columns
  • Support both numerical and textual data generation
  • Preserve statistical properties such as distribution, correlation, and variance
  • Provide an efficient and user-friendly interface
  • Enable data export for use in machine learning and analytical tasks

By addressing these goals, the project seeks to simplify data access and accelerate experimentation in data science workflows.

1.4 Data Sources

Unlike traditional data science projects, this system does not rely on a fixed dataset. Instead, it dynamically generates synthetic data or learns from user-provided datasets depending on the selected generation method.

The project operates with two primary data sources:

  • Synthetic Data (Generated Data): The system generates data from scratch based on user-defined parameters such as the number of rows, columns, and data types. This method does not require any real dataset and is designed for fast, flexible, and privacy-preserving data creation.
  • User-Provided Dataset (Reference Data): For model-based data generation, users can upload a real dataset. The system then learns the statistical structure and relationships within the data using generative models. This enables the creation of synthetic datasets that closely resemble real-world data in terms of distribution and correlations.

This dual approach allows the system to support both rapid data generation and high-fidelity synthetic data modeling, making it suitable for a wide range of use cases.

1.5 Methodology

The system is designed as a modular pipeline that supports multiple synthetic data generation approaches. The overall workflow consists of the following stages:

  • User Input & Configuration: Users define the structure of the dataset, including the number of rows, columns, and data types. Alternatively, users can upload an existing dataset for model-based generation.
  • Data Generation Approach Selection: The system provides two main generation methods: Random (Rule-Based) Generation and Model-Based Generation.
  • Synthetic Data Generation: In random generation, libraries such as Faker are used to create realistic categorical and structured data. In model-based generation, models such as CTGAN, TVAE, CopulaGAN, and Gaussian Copula are used to produce statistically similar datasets.
  • Data Processing & Storage: Generated datasets are processed using Pandas and stored efficiently for further use. The system ensures consistency and usability of the generated data.
  • Quality Evaluation (Optional): Users can evaluate the quality of synthetic data by comparing it with real datasets using various metrics such as statistical similarity, correlation similarity, and machine learning efficacy.

This workflow ensures flexibility, scalability, and usability for both technical and non-technical users.

1.6.1 Random Data Generation Results

Data generation is fast and computationally efficient with each execution producing unique datasets due to stochastic behavior. Logical consistency is maintained through predefined rules, making this method highly suitable for testing, prototyping, and rapid development scenarios.

1.6.2 Model-Based Data Generation Results

Generated data closely matches the statistical properties of real datasets with distributions of numerical and categorical variables being preserved. Relationships between variables (e.g., correlations) are successfully learned, and rare categories are effectively represented.

Although this approach requires additional time for model training, it produces significantly more realistic and analytically useful data.

1.6.3 Quality Evaluation Metrics

To assess the quality of synthetic data, various metrics were used including Statistical Similarity (distribution alignment), Correlation Similarity (relationships between variables), ML Efficacy (model performance trained on synthetic data), and Category Coverage (preservation of categorical values).

The results demonstrate that model-based approaches, particularly CTGAN, provide high-quality synthetic data suitable for machine learning and analytical tasks.

2. System Architecture

The system is designed as a modular and scalable web-based architecture that integrates data generation, model training, evaluation, and user interaction. The architecture consists of the following core components:

  • Frontend Layer: The user interface is built using HTML, CSS, TailwindCSS, and JavaScript. It provides an interactive and responsive environment where users can configure data generation parameters, upload datasets, and visualize outputs.
  • Backend Layer: The backend is developed using Flask, which handles user requests, manages application logic, and coordinates communication between different system components. It acts as the central controller of the system.
  • Data Generation Engine: This module is responsible for generating synthetic data using two approaches: Rule-based generation using predefined logic and libraries, and Model-based generation using generative models trained on real data.
  • Model Training Module: When a dataset is provided, this component trains generative models such as CTGAN, TVAE, CopulaGAN, and Gaussian Copula to learn the underlying data distribution.
  • Evaluation Module: This module compares synthetic and real datasets using statistical and machine learning-based metrics to ensure data quality and reliability.
  • Database Layer: SQLite is used to store user data, configurations, and generated datasets. Flask-SQLAlchemy is used for efficient database management.

This modular design ensures flexibility, maintainability, and scalability, allowing the system to be extended with new models, data types, and evaluation methods.

3. Tech Stack

The project leverages a combination of modern web development and data science technologies:

  • Programming Language: Python
  • Backend Framework: Flask
  • Frontend Technologies: HTML, CSS, TailwindCSS, JavaScript
  • Data Processing & Analysis: Pandas, NumPy
  • Synthetic Data Generation: Faker, SDV (Synthetic Data Vault)
  • Generative Models: CTGAN, TVAE, CopulaGAN, Gaussian Copula
  • Database: SQLite, Flask-SQLAlchemy

This stack enables efficient data processing, flexible model integration, and a user-friendly interface for interaction.

  • Real Data Learning Capability: Allows users to upload real datasets and generate statistically similar synthetic data using generative models
  • Multiple Generative Models: Integrates advanced models such as CTGAN, TVAE, CopulaGAN, and Gaussian Copula
  • Data Quality Evaluation: Provides multiple evaluation metrics including statistical similarity, correlation similarity, ML efficacy, and category coverage
  • User-Friendly Web Interface: Interactive and responsive interface for seamless user experience
  • Fast and Scalable Data Generation: Efficient generation process suitable for both small and large datasets
  • Privacy-Preserving Data Solution: Enables data sharing and experimentation without exposing sensitive information
  • Extensible Architecture: Designed to support future extensions such as time-series data, visualization modules, and API integration
