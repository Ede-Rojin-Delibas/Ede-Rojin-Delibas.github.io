Synthetic Data Generation System

Random and model-based system for generating realistic synthetic datasets

1. Project Explanation

1.1 Introduction

Accessing high-quality datasets remains a critical challenge in machine learning due to privacy regulations, limited availability, and data scarcity. Synthetic data generation offers a powerful solution by creating artificial datasets that preserve statistical properties while maintaining privacy.

1.2 Problem Statement

Many organizations struggle with data limitations due to GDPR/HIPAA regulations, class imbalance, and expensive data collection. Existing solutions are often complex and lack flexibility for customized synthetic data generation.

1.3 Objectives

  • Enable custom dataset generation with configurable parameters
  • Support numerical and categorical data synthesis
  • Preserve statistical properties and data correlations
  • Provide user-friendly interface and fast data generation
  • Ensure privacy-preserving data export capabilities

1.4 Data Sources

The system supports two generation approaches:

  • Rule-Based: Generates data from scratch using Faker library without requiring real datasets
  • Model-Based: Learns from user-provided datasets using generative models (CTGAN, TVAE, CopulaGAN, Gaussian Copula)

1.5 Methodology

  • User Configuration: Define dataset structure (rows, columns, data types)
  • Generation Selection: Choose between random or model-based synthesis
  • Data Generation: Apply selected method to produce synthetic data
  • Processing & Storage: Ensure data consistency and usability
  • Quality Evaluation: Compare synthetic data with real data using multiple metrics

1.6 Evaluation and Results

The system was evaluated based on both performance and data quality across different generation methods.

1.6.1 Random Data Generation Results

Fast and computationally efficient data generation with unique datasets produced on each execution. Ideal for testing, prototyping, and rapid development.

1.6.2 Model-Based Data Generation Results

Generated data closely matches statistical properties of real datasets with preserved distributions and learned correlations between variables. Model-based approaches produce significantly more realistic and analytically useful data compared to random generation.

1.6.3 Quality Evaluation Metrics

Quality assessment compares synthetic and real data using statistical similarity, correlation analysis, ML efficacy, and category coverage metrics to validate synthetic data integrity and usability.

2. System Architecture

  • Frontend: HTML, CSS, TailwindCSS, JavaScript for interactive user configuration
  • Backend: Flask handles requests, logic, and component coordination
  • Generation Engine: Rule-based (Faker) and model-based (CTGAN, TVAE, CopulaGAN, Gaussian Copula) approaches
  • Training Module: Learns data distributions from user-provided datasets
  • Evaluation Module: Compares synthetic and real data using statistical metrics
  • Database: SQLite with Flask-SQLAlchemy for data persistence

3. Tech Stack

  • Backend: Python, Flask
  • Frontend: HTML, CSS, TailwindCSS, JavaScript
  • Data Processing: Pandas, NumPy
  • Generative Models: CTGAN, TVAE, CopulaGAN, Gaussian Copula
  • Data Generation: SDV (Synthetic Data Vault), Faker
  • Database: SQLite, Flask-SQLAlchemy

4. Key Features

  • Dual synthesis approaches (rule-based and model-based)
  • Custom dataset configuration with flexible parameters
  • Multiple generative models (CTGAN, TVAE, CopulaGAN, Gaussian Copula)
  • Data quality evaluation with statistical metrics
  • User-friendly web interface
  • Fast and scalable generation for datasets of any size
  • Privacy-preserving data export
  • Extensible architecture for future enhancements
