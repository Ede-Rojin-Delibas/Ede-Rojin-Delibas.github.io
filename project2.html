Natural Language Processing

Sentiment Analysis

Machine learning-based sentiment classification from social media text

1. Introduction

With the rapid growth of social media platforms, analyzing user-generated content has become essential for understanding public opinion. Sentiment analysis enables businesses and organizations to automatically detect the emotional tone behind text data.

In this project, I developed a machine learning-based sentiment analysis system to classify social media comments into different sentiment categories using natural language processing (NLP) techniques.

2. Problem Statement

Organizations receive a large volume of unstructured textual data from social media, making it difficult to manually analyze user sentiment. The key challenges are:

  • Automatically classifying text into sentiment categories
  • Handling noisy and unstructured text data
  • Achieving reliable performance across different sentiment classes

3. Objective

  • Build a model to classify text into sentiment categories (positive, negative, neutral)
  • Apply NLP preprocessing techniques to improve text quality
  • Compare multiple machine learning models
  • Optimize the best-performing model

4. Data Source

  • Dataset: Social media sentiment dataset (Kaggle)
  • Data type: User-generated text (comments/posts)
  • Target variable: Sentiment label (Positive, Negative, Neutral)

5. Methodology

5.1 Project Planning

  • Dataset selection and exploration
  • Identification of features and target variable
  • Data cleaning and preprocessing pipeline
  • Model selection and training strategy
  • Hyperparameter optimization
  • Evaluation and visualization

5.2 Data Preprocessing & NLP

To improve text quality and model performance, several NLP techniques were applied:

  • Tokenization - Breaking text into individual words
  • Stemming - Reducing words to root form
  • Text tagging - Part-of-speech annotation
  • Stopwords removal - Filtering common words
  • Parsing - Syntactic analysis
  • Removal of unnecessary columns and handling missing data

Libraries used: NLTK (Natural Language Toolkit), Pandas, NumPy, Matplotlib, Seaborn

5.3 Feature Preparation

  • Text data transformed into numerical representations using TF-IDF (Term Frequency-Inverse Document Frequency)
  • Cleaned and structured data for machine learning models

5.4 Model Training

Multiple machine learning algorithms were trained and evaluated:

Model Accuracy
Passive Aggressive Classifier 71.42%
Logistic Regression 63.26%
Random Forest 65%
Support Vector Machine (SVM) 59%
Multinomial Naive Bayes 61.9%

Best-performing model: Passive Aggressive Classifier with 71.42% accuracy

5.5 Hyperparameter Optimization

  • Applied RandomizedSearchCV on the Passive Aggressive Classifier
  • Result: Accuracy improved to 72.78%
  • Improvement was limited, indicating model performance plateau

6. Evaluation & Results

Detailed evaluation using confusion matrix revealed performance across sentiment categories:

Sentiment Accuracy
Positive 83.3%
Negative 81.3%
Neutral 56.4%

Key Insight: The model performs well for positive and negative sentiments, but struggles with neutral classification. Binary-like sentiment (positive vs negative) is easier to classify, while neutral sentiment is inherently ambiguous and harder to model.

7. Business Value

This system can be used to:

  • Monitor brand perception on social media
  • Analyze customer feedback automatically
  • Detect negative sentiment early and take action
